# 섹션별 UX 아이덴티티 개선 계획

**작성일**: 2026-03-15
**상태**: 계획 수립 완료, 구현 예정

---

## 배경

현재 Hub / 통계 분석 / Graph Studio 세 섹션이 동일한 시각적 언어를 사용하여 사용자가 **"지금 어디에 있는지"** 즉시 인지하기 어려움.

### 현재 문제점

| 문제 | 설명 |
|------|------|
| **위치 인지 부재** | 사이드바 하이라이트만으로 현재 섹션 판단 — 헤더, 배경, 색상 모두 동일 |
| **섹션 전환감 없음** | 페이지 이동해도 "이동했다"는 느낌이 없음 |
| **Hub 도착지 불명확** | "빠른 시작" 카드 클릭 시 어디로 가는지 예측 불가 |
| **통계 브라우저 ↔ 분석 플로우** | 메서드 선택 전후의 시각적 변화 부족 |
| **Graph Studio 업로드** | 통계의 데이터 업로드와 거의 동일한 패턴 |

---

## 개선 항목

### S1. 섹션 헤더에 섹션명 + 아이콘 표시 [우선도: HIGH, 난이도: 낮음]

**현재**: Hub와 통계 분석의 헤더가 동일 (우측 아이콘 2개뿐). Graph Studio만 "Graph Studio" 텍스트 있음.

**개선**:
- Hub 헤더 좌측: `🏠 BioHub` 또는 로고
- 통계 헤더 좌측: `📊 통계 분석` + 현재 단계 (예: "Step 1 — 데이터 탐색")
- Graph Studio: 현행 유지 (이미 표시됨)

**구조 메모**:
- 현재 Hub(`/`)와 통계 브라우저(`/analysis`)는 **같은 `AnalysisLayout`** 을 사용함
- 따라서 `AnalysisLayout`에 `sectionVariant`(또는 동등한 prop) 추가가 필요
- `showHub`만으로는 홈 Hub와 `/analysis` 브라우저를 구분할 수 없음

**효과**: 사용자가 헤더만 봐도 현재 위치 즉시 인지

**파일**: `AnalysisLayout.tsx`, `app/page.tsx`, `app/analysis/page.tsx`, `GraphStudioHeader.tsx`

---

### S2. 헤더 상단 컬러 바 (2px accent line) [우선도: HIGH, 난이도: 낮음]

**현재**: 세 섹션 모두 동일한 흰 헤더

**개선**:
- Hub: 블루 (`--primary`, 현행 팔레트)
- 통계 분석: 틸/시안 (과학적, 분석 느낌)
- Graph Studio: 바이올렛/보라 (창작, 도구 느낌)

**구현**: 각 레이아웃 헤더에 `border-top: 2px solid var(--section-accent)` 추가
- `globals.css`에 `--section-accent` CSS 변수 정의
- `[data-section="hub"]`, `[data-section="analysis"]`, `[data-section="graph-studio"]` 형태로 적용
- `AnalysisLayout`은 caller(`app/page.tsx`, `app/analysis/page.tsx`)에서 섹션 값을 받아야 함

**효과**: 무의식적 위치 인지 (시선이 헤더에 닿을 때마다)

---

### S3. 통계 브라우저 → 분석 플로우 전환 시 시각적 변화 [우선도: MEDIUM, 난이도: 중간]

**현재**: 메서드 선택 → 같은 배경에 스테퍼만 추가됨

**개선**:
- 분석 플로우 진입 시 배경에 미세한 틸 틴트 (예: `bg-teal-50/30`)
- 또는 스테퍼 영역 배경 분리 (카드 형태)
- 4단계 스테퍼의 현재 단계 강조를 틸 계열로

**효과**: "분석 모드에 진입했다"는 느낌

**파일**: `AnalysisLayout.tsx`, `StatisticsBrowserHub.tsx`, `FloatingStepIndicator.tsx`

---

### S4. Hub "빠른 시작" 카드에 도착지 명시 [우선도: MEDIUM, 난이도: 낮음]

**현재**: "데이터 업로드", "표본 크기 계산", "데이터 시각화", "Bio-Tools" — 클릭 후 어디로 가는지 불분명

**개선**:
- 각 카드에 작은 뱃지로 **도착지 또는 동작 유형** 표시
- 예시:
  - 데이터 업로드 → `통계 분석 · Step 1`
  - 표본 크기 계산 → `도구 모달`
  - 데이터 시각화 → `Graph Studio`
  - Bio-Tools → `Bio-Tools`
- 또는 카드 좌측에 섹션 컬러 바 (S2와 동일 컬러)

**파일**: `TrackSuggestions.tsx`

---

### S5. Graph Studio 에디터 모드 배경 차별화 [우선도: LOW, 난이도: 낮음]

**현재**: 에디터 모드도 순백 배경

**개선**:
- 에디터 모드 진입 시 캔버스 영역 배경을 미세한 그레이 (`bg-muted/20`) 처리
- IDE/디자인 도구 느낌 강화 (Figma, VS Code 참고)
- 패널 경계선 유지 + 캔버스 영역만 차별화

**효과**: "도구 모드"에 있다는 느낌

---

## 구현 순서

```
Phase 1 (즉시, 1-2시간)
├── S1. 헤더 섹션명 + 아이콘
└── S2. 헤더 accent bar

Phase 2 (다음 주)
├── S4. Hub 카드 도착지 표시
└── S5. Graph Studio 에디터 배경

Phase 3 (여유 시)
└── S3. 통계 플로우 전환 시각 효과
```
