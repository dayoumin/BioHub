# Paper Draft Generation — UI Design Brief

**목적**: 디자인 AI(Figma AI, v0, Stitch 등)에게 전달하여 UI 디자인 생성
**대상 화면**: 2개 (DraftContextEditor 다이얼로그 + PaperDraftPanel)

---

## 프로젝트 컨텍스트

- **제품**: BioHub — 전문가급 통계 분석 웹 플랫폼 (연구자 대상)
- **기술 스택**: Next.js 15 + TypeScript + Tailwind CSS + shadcn/ui (Radix UI)
- **테마**: 라이트/다크 모드 지원, 기본 다크
- **폰트**: system-ui (본문), monospace (통계 수치)
- **디자인 시스템**: shadcn/ui 기반 — Card, Dialog, Tabs, Button, Badge, Input, Label, RadioGroup, Tooltip
- **아이콘**: Lucide React

### 현재 결과 화면 레이아웃 (진입점)

## 화면 1: DraftContextEditor (확인 다이얼로그)

### 역할
"논문 초안" 버튼 클릭 시 **바로 생성하지 않고** 이 다이얼로그를 먼저 표시.
분석에 사용된 변수명/단위/집단명을 자동 감지하여 채워놓고, 사용자가 확인·수정 후 생성.

### 사용 컴포넌트
- `Dialog` (shadcn) — `sm:max-w-[560px]`
- `DialogHeader` + `DialogTitle`
- `Input` (shadcn) — 각 필드의 편집 가능한 값
- `Label` (shadcn) — 필드 라벨
- `RadioGroup` + `RadioGroupItem` — 사후검정 표시 옵션, 언어 선택
- `Button` — 취소(outline), 생성하기(default)
- `Collapsible` 또는 섹션 헤더 — 각 그룹 접기/펼치기 (선택)

### 와이어프레임

```
┌─────────────────────────────────────────────────────┐
│  ✕                                                  │
│                                                     │
│  논문 초안 생성                                       │
│  분석 결과를 학술 논문 형식으로 변환합니다.              │
│  자동 감지된 정보를 확인하고 필요시 수정하세요.          │
│                                                     │
│  ─────────────────────────────────────────────────── │
│                                                     │
│  변수 표시명                                         │
│  ┌──────────────────────────────────────────────┐   │
│  │  body_len    →    [체장              ]       │   │
│  │  weight      →    [체중              ]       │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  단위                                               │
│  ┌──────────────────────────────────────────────┐   │
│  │  body_len    →    [cm                ]       │   │
│  │  weight      →    [g                 ]       │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  집단명                              ← 조건부 표시   │
│  ┌──────────────────────────────────────────────┐   │
│  │  M           →    [수컷              ]       │   │
│  │  F           →    [암컷              ]       │   │
│  └──────────────────────────────────────────────┘   │
│                                                     │
│  종속변수                                           │
│  [체장                                          ]   │
│                                                     │
│  연구 맥락 (선택)                                    │
│  [양식 어류의 성별에 따른 성장 차이 비교           ]   │
│  ℹ️ Methods 도입부와 Discussion에 활용됩니다          │
│                                                     │
│  ─────────────────────────────────────────────────── │
│                                                     │
│  사후검정 표시           ← postHoc 없으면 숨김        │
│  ◉ 유의한 쌍만   ○ 전체                              │
│                                                     │
│  언어                                               │
│  ◉ 한글   ○ English                                 │
│                                                     │
│  ─────────────────────────────────────────────────── │
│                                                     │
│                         [취소]   [📝 생성하기]        │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### 디자인 요구사항

| 항목 | 상세 |
|------|------|
| **레이아웃** | 2-column 그리드: 좌=원본 코드명(muted, monospace), 우=편집 Input |
| **자동 감지 표시** | 원본 코드명은 `text-muted-foreground font-mono text-sm`으로 읽기 전용 |
| **Input 스타일** | shadcn Input, 값이 자동 채워져 있으면 일반, 비어있으면 `border-dashed` |
| **조건부 섹션** | `groupStats` 없으면 "집단명" 섹션 전체 숨김. `postHoc` 없으면 "사후검정" 숨김 |
| **재진입 프리필** | 이전 생성 결과가 있으면 저장된 `DraftContext`로 모든 필드 프리필 |
| **연구 맥락** | textarea 1줄 (auto-resize), 하단에 muted 도움말 텍스트 |
| **언어 토글** | RadioGroup 가로 배치, 기본값 "한글" |
| **버튼** | 취소=outline variant, 생성하기=default variant with FileText 아이콘 |
| **스크롤** | 변수가 많으면 다이얼로그 내부 스크롤 (max-h-[70vh]) |

### 상태 변형 (디자인 시 고려)

1. **기본 상태**: 변수 2-3개, 집단 2개, 사후검정 없음 → 컴팩트 (스크롤 불필요)
2. **많은 변수**: 변수 8-10개 → 내부 스크롤 활성화
3. **집단 없는 분석** (상관분석, 회귀분석): 집단명 섹션 숨김, 사후검정 숨김
4. **재진입**: 모든 필드 프리필 + 상단에 "이전 설정을 불러왔습니다" 알림 배지

## 화면 2: PaperDraftPanel (결과 패널)

### 역할
생성된 논문 초안을 섹션별 탭으로 표시. 각 섹션 개별 복사 + 전체 복사 지원.
Methods/Results/Captions는 즉시 표시, Discussion은 별도 LLM 호출로 스트리밍.

### 사용 컴포넌트
- `Card` (shadcn) — 전체 래퍼
- `Tabs` + `TabsList` + `TabsTrigger` + `TabsContent` — 4개 섹션 탭
- `Button` — 복사(ghost+icon), 전체 복사(outline), AI 생성(default), 언어 토글, 재생성
- `Badge` — 언어 표시, 생성 시간, 모델명
- `Tooltip` — 복사 성공 피드백
- `Skeleton` — Discussion 스트리밍 로딩

### 배치 위치
결과 화면(ResultsActionStep) 내에서 **AI 해석 카드 아래** 또는 **별도 섹션**으로 표시.
다이얼로그가 아닌 **인라인 패널** (접기/펼치기 가능).

### 와이어프레임

```
┌─────────────────────────────────────────────────────────┐
│  📄 논문 초안                                            │
│                                                         │
│  ┌─────────────────────────────────┐  [한글 ↔ EN]       │
│  │ [전체 복사]  [다시 생성]          │  2024-03-12 생성   │
│  └─────────────────────────────────┘                    │
│                                                         │
│  ┌─────────┬─────────┬──────────┬────────────┐          │
│  │ Methods │ Results │ Captions │ Discussion │          │
│  ├─────────┴─────────┴──────────┴────────────┤          │
│  │                                           │          │
│  │  (현재 탭: Methods)                        │          │
│  │                                           │          │
│  │  두 집단 간 체장 차이를 검증하기 위해        │          │
│  │  독립표본 t-검정을 실시하였다. 분석에 앞서    │          │
│  │  Shapiro-Wilk 검정으로 정규성을             │          │
│  │  (W = 0.96, p = .712; W = 0.94, p = .534), │          │
│  │  Levene 검정으로 등분산성을                  │          │
│  │  (F = 1.23, p = .277) 확인하였으며,         │          │
│  │  모든 가정이 충족되었다.                     │          │
│  │  유의수준은 α = .05로 설정하였다.            │          │
│  │  통계 분석은 BioHub를 사용하여 수행하였다.    │          │
│  │                                           │          │
│  │                               [📋 복사]   │          │
│  └───────────────────────────────────────────┘          │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 디자인 요구사항

| 항목 | 상세 |
|------|------|
| **전체 래퍼** | `Card` with `border-blue-200 dark:border-blue-800/50` (학술 느낌 — AI 해석의 violet과 구분) |
| **헤더 영역** | FileText 아이콘 + "논문 초안" 타이틀 + 우측에 언어 토글 + 생성시간 Badge(muted) |
| **액션 버튼** | "전체 복사"(outline, Copy 아이콘) + "다시 생성"(ghost, RefreshCw 아이콘) |
| **탭 스타일** | shadcn `Tabs` default — 4개 탭 균등 배분 |
| **텍스트 렌더링** | `prose dark:prose-invert` 마크다운 렌더링 (이탤릭 통계기호 지원) |
| **복사 버튼** | 각 탭 우하단 ghost 버튼, 클릭 시 체크마크로 1.5초 전환 (Tooltip "복사됨") |
| **언어 토글** | 작은 토글 버튼 (한글/EN), 전환 시 템플릿 섹션 즉시 재생성 |
| **Discussion 상태** | idle(CTA 버튼) → streaming(타이핑 애니메이션) → done(텍스트+모델명) → error(경고+재시도) |
| **접기/펼치기** | 패널 전체를 접을 수 있는 Collapsible (다른 결과도 함께 볼 수 있도록) |

### 상태 변형 (디자인 시 고려)

1. **첫 생성 직후**: Methods/Results/Captions 즉시 표시, Discussion은 idle
2. **Discussion 스트리밍**: 4번째 탭에 타이핑 애니메이션 + 진행 표시
3. **전체 완료**: 모든 탭에 콘텐츠, "전체 복사" 활성화
4. **히스토리에서 복원**: 저장된 초안 즉시 표시, 상단에 "저장된 초안" Badge
5. **에러 상태**: Discussion만 에러 (나머지 3개는 정상)

```
결과 화면 하단 액션 바

  ↓ "논문 초안" 항목 선택
```
