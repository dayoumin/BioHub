# 사용자 관점 논문 지원 워크플로우 (User Journey)

**Status**: 기획 단계
**작성일**: 2026-03-27
**목적**: "문헌·동향"과 "논문 작성" 기능을 개발자 관점이 아닌 사용자 관점(UI/UX 흐름)에서 명확히 설계하여 구현 방향을 통일함.

---

## 1. 전체 여정 요약 (The Researcher's Path)

BioHub의 논문 지원 시스템은 사용자가 다음과 같은 4단계를 거치도록 설계됩니다.

1. **발견 (Discover)**: `/literature`에서 내 연구 분야의 선행 연구와 트렌드를 파악
2. **수집 (Collect)**: 중요한 논문을 "내 프로젝트"의 참고문헌 폴더에 저장
3. **생산 (Analyze)**: `/` (Smart Flow)에서 내 데이터를 넣어 통계/생물학적 분석을 수행
4. **작성 (Write)**: `/papers`에서 수집한 참고문헌과 분석 결과를 합쳐 논문 초안(Methods, Results, References) 완성

이 중 (3) 생산(통계)은 이미 BioHub의 핵심으로 완성형에 가깝습니다. 이번 계획은 (1), (2), (4)를 어떻게 하나의 "논문 지원" 파이프라인으로 연결할지에 초점을 맞춥니다.

---

## 2. 사용자 여정 상세 설계

### Phase 1: 발견 및 수집 (문헌·동향 페이지)

**사용자 페르소나**: K연구원, 새로운 수산 생물(예: 넙치) 바이러스 논문을 쓰기로 결심함.

#### Scene 1: 연구 동향 파악
1. K연구원이 좌측 사이드바에서 **[문헌·동향]** 메뉴를 클릭합니다.
2. 현재 활성화된 화면은 **[연구동향]** 탭입니다.
3. K연구원은 `Paralichthys olivaceus` (넙치)와 `virus` 키워드를 "나의 관심 키워드"로 등록합니다.
4. 마치 뉴스 피드처럼, 최근 1주/1달간 해당 키워드로 출판된 전 세계 논문 목록이 타임라인 형태로 뜹니다.

#### Scene 2: 선행 연구 집중 검색
1. 특정 주제를 집중적으로 파헤치기 위해 **[문헌검색]** 탭으로 이동합니다.
2. 검색창에 구체적인 키워드를 넣습니다. "어떤 데이터베이스에서 찾을까요?" 옵션이 보입니다.
   - [x] OpenAlex (국제 논문)
   - [x] PubMed (의생명)
   - [x] 국회도서관 (국내 논문)
   - [ ] GBIF/OBIS (표본 기록 - 이번엔 체크 해제)
3. 검색을 누르자 3개 DB에서 동시에 막대기가 차오르며 논문이 쏟아집니다. (병렬 검색 UX)
4. 결과 목록을 훑어보다가, 자신의 연구와 직결되는 논문 5편을 발견합니다.

#### Scene 3: 인용 폴더에 담기 (Collect)
1. 논문 카드 우측에 있는 **[+ 인용에 추가]** 버튼을 클릭합니다.
2. 우측 하단 토스트: `[바이러스 연구] 프로젝트의 인용 목록에 추가되었습니다.`
3. (선택) 원문이 필요하면 논문의 DOI 링크를 클릭해 출판사 사이트로 날아가 PDF를 다운로드한 후, 그것을 **NotebookLM**에 던져넣고 요약을 읽습니다. (BioHub 밖의 행위)

---

### Phase 2: 데이터 생산 (통계 분석 페이지)

(이 과정은 기존 BioHub 워크플로우를 그대로 따릅니다)

1. K연구원은 실험실에서 얻은 넙치 감염 데이터를 CSV로 준비합니다.
2. 활성 프로젝트가 `[바이러스 연구]`로 선택된 상태에서 통계 분석을 돌립니다. (t-test, ANOVA 등)
3. 결과가 만족스럽게 나오면 자동화된 "저장" 기능이 작동합니다.

---

### Phase 3: 논문 초안 작성 (논문 작성 페이지)

**가장 혁신적인 경험이 일어나는 곳입니다.** K연구원이 분석과 선행연구 조사를 모두 마치고 논문을 쓸 준비가 되었습니다.

#### Scene 1: 결과 정리 시작
1. 좌측 사이드바에서 **[논문 작성]** 메뉴를 클릭합니다.
2. 화면에 현재 `[바이러스 연구]` 프로젝트에 모인 "재료"들이 보여집니다.
   - [분석 기록] 3건 (t-test 등)
   - [참고문헌] 5건 (문헌검색에서 담아둔 것)
   - [그래프] 2건

#### Scene 2: 문서 조립 (Document Blueprint)과 NotebookLM 합치기
1. **[새 논문 초안 만들기]** 를 클릭합니다.
2. 화면이 큰 캔버스 형태로 바뀌며, 논문의 목차(Introduction, Methods, Results, Discussion, References)가 보입니다.
3. **Methods & Results (BioHub 데이터)**: 저장해둔 [분석 기록 3건]을 드래그 앤 드롭으로 캔버스에 끌어다 놓습니다.
   - 마법처럼 "Student's t-test was used to compare..." (Methods)와 "There was a significant difference..." (Results), 그리고 APA 7th 저널 표가 자동으로 생성되어 문서에 박힙니다.
4. **Introduction & Discussion (NotebookLM 데이터 합류 단계)**:
   - BioHub는 이 부분을 스스로 창작하지 않습니다. 대신 K연구원이 **NotebookLM에서 조사하고 만들어둔 자료를 가져와 넣을 수 있는 공간**을 제공합니다.
   - K연구원이 NotebookLM에서 정리해둔 개요 텍스트, 선행 연구 요약을 복사해 `[Introduction]` 빈칸에 붙여넣습니다.
   - NotebookLM에서 생성한 "바이러스 감염 경로 다이어그램" 이미지를 드래그 앤 드롭하여 캔버스에 그림 파일(Figure)로 끼워 넣습니다.
   - 텍스트를 붙여넣다가, 저장해둔 논원 1번 항목(문헌검색에서 찾았던 것)을 클릭하면 텍스트 에디터 안에 `(Kim et al., 2025)` 형식으로 **인라인 인용이 삽입**됩니다.

#### Scene 3: 자동 참고문헌 완성
1. 인라인 인용 `(Kim et al., 2025)`을 본문에 삽입하는 순간, 논문 맨 밑의 **[References]** 섹션에 APA 서식으로 풀 레퍼런스가 자동으로 방금 전 타이핑됩니다.
   - `Kim, J., Lee, S. (2025). Viral diseases of flounder...`
2. 남은 인용들도 삽입하며 초안을 완성합니다.

#### Scene 4: 내보내기
1. 모든 정리가 끝났습니다. **[내보내기]** 를 클릭합니다.
2. "Word(.docx)로 내보내기"를 선택하자 모든 텍스트, 이탤릭체 p-value 기호, APA 표, 참고문헌 목록이 다 채워진 완벽한 뼈대 문서가 다운로드됩니다.
3. 만약 Zotero를 써서 추가 작업을 하고 싶다면, 참고문헌만 따로 `BibTeX 다운로드`를 누르면 됩니다.

## 3. UI/UX 구현 과제 (무엇을 어떻게 만들 것인가?)

사용자 여정을 실현하기 위해 우리가 구현해야 할 UI/UX 컴포넌트들입니다.

### 3-1. 문헌·동향 (`/literature`)
- **수집 장바구니 패턴**: E-commerce의 장바구니처럼, 검색 결과 각 항목에 "담기/빼기" 토글이 있어야 합니다.
  - 전제: 활성 프로젝트(Active Project)가 반드시 존재하거나, 없으면 "개별 작업/장바구니"용 임시 공간 필요.
  - UI 구성: 사이드바나 화면 우측 상단에 `[인용 카트: 3건]` 같은 UI 힌트가 필요합니다.
- **스트리밍 프로그레스바 (UX 핵심)**:
  - OpenAlex는 1초, 국회도서관은 3초가 걸릴 수 있습니다. "기다려주세요" 스피너만 돌리면 답답합니다.
  - 각 소스별 완료 상태를 보여주는 인디케이터 필요. (`OpenAlex 완료, PubMed 검색 중...`)

### 3-2. 논문 작성 (`/papers`)
- **인용 사이드패널 (Citation Picker)**:
  - 에디터 옆에 띄워 두고 검색(필터)할 수 있는 인용 목록 바. 클릭하면 에디터 커서 위치에 `(Author, Year)` 형태로 삽입.
- **자동 References 동기화 (Reactive UX)**:
  - 본문에 인라인 인용이 삽입되었는지(또는 삭제되었는지)를 감지하여 맨 하단 References 목록을 실시간으로 다시 그립니다.
- **분할 뷰 / Document Assembly**:
  - 기존에 1건의 분석 결과만 보던 것을, "캔버스(좌측 문서)"와 "팔레트(우측 재료: 내 분석들, 내 인용들)" 형태의 UI로 확장해야 합니다. (기존 PLAN-DOCUMENT-ASSEMBLY.md 기획 참고 — 삭제됨, 핵심 방향은 이 문서에 통합)

## 4. 로드맵에 따른 점진적 전달 계획

모든 걸 한 번에 만들 수 없으므로, 사용자 가치를 최우선으로 끊어 구현합니다.

**Stage 1. 발견 (Discovery)** - Phase A, B
- 통합검색 UI와 워커(Workers) API 연동.
- 연구동향 UI 구현.
- *(장바구니/인용폴더 기능은 생략 혹은 최소한의 로컬스토리지 임시 저장만 구현)*

**Stage 2. 수집 (Collection)** - Phase E
- 프로젝트 스토리지에 `LiteratureItem` 타입을 "인용 엔티티"로 넣을 수 있게 수정.
- 문헌검색 화면에 "인용 카트(프로젝트 저장)" 기능을 활성화.

**Stage 3. 작성 (Assembly)** - Paper Draft Generation
- `/papers` 페이지를 Document Blueprint 캔버스 형태로 개편.
- 우측에 방금 전 Stage 2에서 구축한 "인용 엔티티 패널" 제공.
- 텍스트 박스에 삽입 및 References 자동 생성기 렌더링.

## 5. 기존 논문 작성 기획서와의 정렬

기존의 `PLAN-DOCUMENT-ASSEMBLY.md`와 `PLAN-PAPER-DRAFT-GENERATION.md`(둘 다 삭제됨)는 "통계 분석 결과를 어떻게 논문으로 바꿀까"라는 **분석 중심의 기획**이었습니다.

이번 `PLAN-USER-JOURNEY-LITERATURE-PAPERS.md`는 여기에 **"선행 연구와 외부 문헌 데이터를 어떻게 합류시킬까"**라는 관점을 추가하여, 두 거대한 기능(문헌 검색과 통계 분석)이 만나는 교차로로 `/papers` 페이지를 재정의합니다.
