# UI/UX 전면 재설계 계획

**작성일**: 2026-03-06
**목표**: 전문적 신뢰감 + 2026 AI 시대 디자인 + 전체 플랫폼 일관성
**데모 목표**: 통계 플로우(Hub→Step 4) + 학명 유효성 검증을 일부 사용자에게 공개

---

## 1. 배경 및 문제 진단

### 현재 앱의 구조적 문제

| 문제 | 현상 |
|------|------|
| **3개의 서로 다른 앱쉘** | Smart Flow / Graph Studio / 레거시 페이지가 각각 다른 UI 언어 |
| **Graph Studio 이중 헤더** | 공통 Header + 전용 GraphStudioHeader 겹침 |
| **도구 간 이동 불가** | Smart Flow ↔ Graph Studio 전환 수단 없음 |
| **사이드바 Graph Studio 버튼 disabled** | 동작하는 페이지인데 "준비 중" 표시 |
| **채팅창이 주인공** | 정확도가 중요한 통계 도구에서 AI 채팅이 진입점 → 불안감 |
| **스테퍼 floating** | 콘텐츠 위에 겹쳐 떠있어 레이아웃과 분리된 느낌 |
| **색상 노이즈** | 형광 아이콘, 스텝별 다른 색상, 통계 유의성이 장미색(직관 반대) |
| **사이드바 흰색** | 배경과 구분 안 됨 — 시각적 계층 없음 |
| **다크모드 파란색 소멸** | `.dark`에서 primary가 무채색(0.9 L, 0 C) — 브랜드 없음 |

### 디자인 철학

> **"조용하지만 강력한 (Calm Intelligence)"**
> 인터페이스는 배경으로 후퇴하고, 데이터와 숫자가 주인공이 된다.

참조: Perplexity(AI 정보 도구) + Stripe(데이터 신뢰성) 하이브리드
— R Studio/SPSS 같은 구식 IDE 스타일이 아닌, 2026년 AI 네이티브 전문 도구

## 2. 플랫폼 구조 설계

### 2-1. 사이드바 (전체 플랫폼 뼈대)

```
[접힘 — 기본 44px]          [펼침 — 220px, 사용자 토글]
┌────┐                      ┌──────────────────────┐
│ ≡  │  ← 토글 버튼          │ ≡  NIFS 통계 분석    │  ← 앱 타이틀 (통일)
├────┤                      ├──────────────────────┤
│ 🏠 │  홈                  │ 🏠  홈               │
│ 📊 │  통계 분석            │ 📊  통계 분석         │
│ 📈 │  Graph Studio        │ 📈  Graph Studio      │
│ 🧬 │  Bio-Tools           │ 🧬  Bio-Tools    예정 │
│ 📄 │  논문 지원            │ 📄  논문 지원    예정 │
│ 🔬 │  학명 유효성 검증     │ 🔬  학명 유효성 검증  │
├────┤                      ├──────────────────────┤
│ ⭐ │  My Menu (예약)       │ ⭐  My Menu      예정 │
├────┤                      ├──────────────────────┤
│ ⚙️  │  설정 (하단 고정)     │ ⚙️   설정             │
└────┘                      └──────────────────────┘
```

**동작 방식**
- 기본: 접힘(44px) — 아이콘 + 툴팁
- 토글(≡ 버튼): 펼침(220px) — 아이콘 + 섹션명
- 상태 저장: localStorage 영속
- 활성 섹션: 파란 좌측 테두리(2px) + 아이콘 강조

**앱 타이틀 통일**: 사이드바 헤더, Hub 헤더, 브라우저 탭 모두 **"NIFS 통계 분석"** 으로 통일
(현재 "NIFS 통계 분석 플랫폼" / "NIFS 통계 분석" 혼재 → 단일화)

**My Menu 개념 (향후 구현)**
사용자마다 자주 쓰는 도구가 다름 → 즐겨찾기 핀. 통계의 "빠른 분석" 칩처럼 개인화된 빠른 접근.
구현 전까지 공간만 예약(divider로 분리, 빈 상태).

### 2-2. 컨텍스트 인식 헤더 (섹션별 상이)

각 섹션이 자체 헤더를 갖는다. 공통 `ConditionalHeader`는 제거하고 섹션별 헤더로 대체.

| 섹션 | 헤더 구성 |
|------|-----------|
| **홈** | 앱 로고 / 알림 / 설정 |
| **통계 분석** | 현재 분석명 / **4단계 스테퍼** / 진행 바 / 저장 |
| **Graph Studio** | `Graph Studio` / 차트 종류 / Undo·Redo / AI / Export |
| **Bio-Tools** | `Bio-Tools` / 현재 도구명 |
| **논문 지원** | `논문 지원` / 현재 섹션명 |
| **학명 검증** | `학명 유효성 검증` / 검색 상태 표시 |

> **섹션 이탈 정책**: 분석 진행 중 사이드바로 다른 섹션 이동 시
> - Step 1(데이터 탐색) 이전: 경고 없이 이동 (손실 없음)
> - Step 2 이후 진행 중: "분석이 자동 저장됩니다" 토스트 + 히스토리에 draft 저장 후 이동
> - 결과 화면: 경고 없이 이동 (이미 완료)

### 2-3. 4단계 스테퍼 재설계

**현재 문제**: `FloatingStepIndicator` — 콘텐츠 위에 fixed로 겹침, pill 모양, 색상 혼재

**변경**: 헤더 하단 고정 영역으로 통합

```
┌──────────────────────────────────────────────────────┐
│  독립표본 t-검정 분석                         [저장]  │  ← 타이틀 행
├──────────────────────────────────────────────────────┤
│  ① 탐색   ② 방법   ③ 변수 ●   ④ 분석               │  ← 스텝 행 (기존 레이블 유지)
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━░░░░░░░░                │  ← 진행 바
└──────────────────────────────────────────────────────┘
```

**스텝 레이블**: 기존 코드 `stepShortLabels`(탐색/방법/변수/분석) 유지
(Terminology 시스템과 연동 — `useTerminology()` 훅 사용 필수)

- floating 완전 제거 (z-index 겹침 없음)
- 숫자① + 텍스트 + 하단 진행 바 (75% fill)
- 완료: 체크(✓) + 흐린 텍스트
- 현재: 파란색 강조
- 미래: 회색
- 클릭: **완료된 스텝만** 이전으로 이동 가능 (미래 스텝 건너뛰기 불가)

## 3. 디자인 시스템

### 3-1. 색상 (OKLCH 교정)

현재 OKLCH 시스템 유지, 아래 값만 교정:

```css
/* ── 사이드바 ─────────────────────────────────────── */
--sidebar:            oklch(0.13 0.02 260);  /* 현재: 0.98 흰색 → 짙은 네이비 */
--sidebar-foreground: oklch(0.82 0.01 260);  /* 부드러운 흰색 */
--sidebar-accent:     oklch(0.20 0.03 260);  /* hover 상태 */
--sidebar-border:     oklch(0.22 0.02 260);  /* 내부 구분선 */

/* ── 배경 / 서피스 ────────────────────────────────── */
--background:  oklch(0.985 0.003 250);       /* 순백→미세 쿨톤 */
--card:        oklch(1 0 0);                 /* 패널은 순백 */
--border:      oklch(0.91 0.008 260);        /* 쿨한 경계선 */

/* ── 브랜드 ───────────────────────────────────────── */
--primary:     oklch(0.52 0.20 260);         /* 더 선명한 Science Blue */

/* ── 통계 유의성 (장미색→초록으로 변경) ────────────── */
/*
 * 변경 근거: 의학·생물학 관례에서 초록 = 통과/유효/유의(p<0.05),
 * 회색 = 중립/비유의(p≥0.05). 장미/빨강은 "경고/위험" 연상으로
 * 유의한 결과가 나쁜 것처럼 오해를 유발함.
 * 참조: 임상시험 결과 보고, 생태학 논문 관례.
 */
--stat-significant:        oklch(0.50 0.14 145);  /* 장미색→초록 */
--stat-highly-significant: oklch(0.44 0.16 145);  /* 더 진한 초록 */
--stat-non-significant:    oklch(0.55 0 0);        /* 그대로 회색 */

/* ── 다크모드 (파란 accent 복원) ─────────────────── */
/* .dark 블록에서 아래 값으로 교체 */
--primary:     oklch(0.65 0.18 260);         /* 현재: 0.9 무채색 → 파란색 */
--sidebar:     oklch(0.08 0.02 260);         /* 더 진한 네이비 */
```

**색상 대비 검증 (WCAG AA 기준 4.5:1)**

구현 후 아래 조합을 [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) 또는 브라우저 DevTools로 실측 필요:

| 조합 | 예상 | 확인 필요 |
|------|------|-----------|
| 사이드바 배경(0.13) + 아이콘(0.82) | 충분 | Phase 1 완료 후 |
| primary(0.52) 버튼 위 흰 텍스트 | 경계 | Phase 0 완료 후 |
| stat-significant(초록) 단독 텍스트 | 확인 필요 | Phase 0 완료 후 |

### 3-2. 타이포그래피

```
UI 텍스트:  Pretendard Variable
            — 한국어 최적화, Toss·카카오 사용 폰트
            — 기존 Geist Sans 대체

통계 숫자:  JetBrains Mono
            — t = 10.96, p < .001, Cohen's d = 4.00 전용
            — 숫자가 "읽히는" 것이 아니라 "보이는" 것

스케일:     12 / 14 / 16 / 18 / 24 / 32 / 48px
굵기:       400(본문) / 500(레이블) / 600(소제목) / 700(통계값 headline)
```

**설치 방법**

Pretendard는 Google Fonts 미등재 폰트이므로 `next/font/google` 사용 불가.

```bash
# 방법 A — npm 패키지 (권장, 오프라인 빌드 가능)
pnpm add @fontsource-variable/pretendard
```
```ts
// layout.tsx 또는 globals.css에서
import '@fontsource-variable/pretendard'
// CSS: font-family: 'Pretendard Variable', sans-serif;
```

```css
/* 방법 B — CDN @import (외부망 의존, globals.css 상단) */
@import url('https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/variable/pretendardvariable-dynamic-subset.min.css');
```

JetBrains Mono는 Google Fonts 등재 폰트이므로 next/font 사용 가능:

```ts
// layout.tsx
import { JetBrains_Mono } from 'next/font/google'

const jetbrainsMono = JetBrains_Mono({
  subsets: ['latin'],
  variable: '--font-mono',
})
```

### 3-3. 컴포넌트 원칙

**카드**
```
현재: shadow + 색상 배경 + 두꺼운 border → 시끄러움
변경: border 1px + 흰 배경               → 조용함
강조: 색상 면적이 아닌 타이포그래피로
```

**통계 결과 숫자 표시**
```
t = 10.96                 ← 48px JetBrains Mono Bold
Independent t-Test        ← 13px muted, 아래 배치

p < .001     Cohen's d = 4.00
● 유의함     ● 매우 큰 효과   ← 14px, 색상 점(dot) + 텍스트
```

**색상 사용 원칙**
- 의미 있을 때만 (유의=초록 dot, 비유의=회색 dot)
- 형광 아이콘 제거 → 단색 Lucide 아이콘
- Accent(파란색) 하나만 — CTA, 활성 상태

**Terminology 시스템 준수**
신규 컴포넌트(`AppSidebar`, `SectionHeader`, `AnalysisFlowHeader`) 모두
`useTerminology()` 훅을 통해 텍스트를 관리. 하드코딩 한국어 금지.

## 4. 각 화면 재설계

### 4-1. Hub (홈) — 워크스페이스 대시보드

**현재 문제**: 마케팅 Hero + 채팅창이 중앙 주인공 → 통계 도구 같지 않음

**변경 방향**: 도구 진입 대시보드 + 최근 작업

도구 카드 순서: 사이드바 순서와 일치 (홈 제외)

```
┌──────────────────────────────────────────────────────┐
│  NIFS 통계 분석                           [설정]      │  ← Hub 헤더
├──────────────────────────────────────────────────────┤
│                                                       │
│  오늘 무엇을 분석하시겠어요?                           │  ← 간결한 인사
│                                                       │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐  │
│  │  📊           │ │  📈           │ │  🧬           │  │  ← 사이드바 순서와 일치
│  │  통계 분석    │ │  Graph Studio │ │  Bio-Tools    │  │
│  │  AI 추천 분석 │ │  논문 품질 차트│ │  (준비 중)    │  │
│  └──────────────┘ └──────────────┘ └──────────────┘  │
│  ┌──────────────┐ ┌──────────────┐                   │
│  │  📄           │ │  🔬           │                   │
│  │  논문 지원    │ │  학명 유효성  │                   │
│  │  (준비 중)    │ │  검증         │                   │
│  └──────────────┘ └──────────────┘                   │
│                                                       │
│  최근 작업 ────────────────────────────────────────  │
│  📊 독립표본 t-검정  어제  [계속하기 →]               │
│  📈 막대 차트 편집   3일전 [계속하기 →]               │
└──────────────────────────────────────────────────────┘
```

- Hero 섹션(`"데이터에서 인사이트까지"`) 제거
- 채팅 입력창 Hub에서 제거
- 도구 카드: 아이콘 단색, border만, 설명 1줄

### 4-2. 통계 분석 플로우 (Step 1~4)

**채팅 입력창 위치**: Hub에서 제거 후 **Step 1(탐색) 화면 하단**에 보조 도구로 배치.
"어떤 분석을 해야 할지 모르겠다면 AI에게 물어보세요" 형태의 접기 가능한 패널.
Step 2 이후에는 노출하지 않음 (방법 이미 선택된 상태).

**Step 1: 데이터 탐색**
- 파일 업로드 dropzone (비주얼 정제)
- 업로드 후: 데이터 미리보기 테이블 자동 표시
  - **최대 100행만 렌더링** (전체 행 수는 "30행 × 6열" 형태로 별도 표시)
  - 대용량(>10,000행) 경고 배지 표시
- 컬럼 요약 (숫자형/범주형 자동 감지 배지)
- 하단: AI 보조 채팅 패널 (접기 가능)

**Step 2: 방법 선택**
- AI 추천 카드: 형광 제거, border + 흰 배경
- 추천 근거 텍스트: 더 구체적, 학문적 표현
- "방법 변경" 링크: 전체 목록 접근

**Step 3: 변수 선택**
- 좌(독립변수) / 우(종속변수) 2열 레이아웃 유지
- 색상: info(파랑)/success(초록)/highlight(보라) 시맨틱 유지
  ※ success(초록)이 "집단 변수"와 "통계 유의성" 양쪽에 쓰이나,
  사용 맥락이 달라 혼동 없음 (변수 선택 단계 vs 결과 단계)
- 카드 테두리 조용하게 (현재 배경 색상 → border 색상으로)

**Step 4: 결과**
```
통계 결론 Hero
─────────────────────────────────────────────────
✓ 통계적으로 유의한 차이가 있습니다
  Independent Samples t-Test                3월 5일 04:09

  t = 10.96              ← 48px, JetBrains Mono Bold
  ─────────────────────
  df = 28               ← 13px muted

  p < .001    Cohen's d = 4.00
  ● 유의함    ● 매우 큰 효과

AI 해석
─────────────────────────────────────────────────
  [결론] 수컷이 암컷보다 체중이 유의미하게 높습니다.
  [수치] t(28) = 10.96, p < .001, d = 4.00
  [의미] 효과 크기가 매우 커 실질적 차이가 명확합니다.
```

### 4-3. Graph Studio

- 이중 헤더 제거 (`ConditionalHeader`에서 `/graph-studio` 제외)
- 동일 `AppSidebar` 적용
- `GraphStudioHeader`에서 `← 홈` 링크 제거 (사이드바가 내비게이션 역할 담당)
