# Phase 4: 사용자 관점 (UX) 테스트

> 실제 사용자 시나리오 기반 — **통계 분석 UX**와 **그래프/시각화 UX** 각각 검증

## 목표

개발자/QA 관점이 아닌 **실제 사용자 입장**에서의 흐름을 검증한다:
- "처음 방문한 연구자가 t-검정을 끝까지 할 수 있는가?"
- "그래프 색상을 바꾸고 논문용 이미지를 다운로드할 수 있는가?"
- "에러가 발생해도 사용자가 복구할 수 있는가?"

---

## Part A: 통계 분석 UX (ux/statistics-ux.spec.ts)

### 4A.1 첫 방문 사용자 시나리오 (@critical)

```
TC-4A.1.1: 완전 초보 — 데이터만 가진 연구자
  시나리오:
    1. / 페이지 접근 (첫 방문)
    2. "데이터 업로드" 카드 인식 → 클릭
    3. CSV 파일 드래그 앤 드롭 (또는 파일 선택)
    4. 데이터 미리보기 확인 → "다음" 클릭
    5. "AI 추천" 탭에서 자연어 질문 입력
       예: "두 그룹의 평균이 차이가 있는지 알고 싶어요"
    6. 추천 메서드 카드 확인 → "이 방법으로" 클릭
    7. 변수 자동 할당 → "분석 실행" 클릭
    8. 결과 확인: 통계량 + p-value + 해석 텍스트
    9. "내보내기" → DOCX 다운로드
  검증:
    - 각 단계에서 사용자가 다음 행동을 알 수 있는 UI 단서 존재
    - 모든 버튼이 적절한 시점에 활성화
    - 결과 화면에 해석(한국어) 텍스트 존재

TC-4A.1.2: 통계 전문가 — 메서드 직접 선택
  시나리오:
    1. / 페이지 → 업로드 카드 클릭
    2. paired-t.csv 업로드
    3. "직접 선택" 탭 → "대응표본" 검색 → 선택
    4. 변수 수동 할당 (pre → 사전, post → 사후)
    5. 분석 실행 → 결과 확인
    6. 효과크기(Cohen's d) 확인
    7. 가정 검정(정규성) 확인
    8. 결과를 엑셀로 내보내기
  검증:
    - 직접 선택 → 변수 선택 → 결과까지 3분 이내
    - 전문 통계 용어(자유도, 신뢰구간) 표시

TC-4A.1.3: QuickAnalysisPills 사용
  시나리오:
    1. Hub 화면의 Quick Pills (t-test, ANOVA 등) 클릭
    2. Quick 모드 진입 → Step 1(업로드) 직행
    3. 데이터 업로드 → 자동으로 해당 메서드 선택됨
    4. 변수 할당 → 분석
  검증:
    - 빠른 분석 모드에서 Step 2 건너뛰기 확인
    - 총 클릭 수 5회 이하로 결과 도달
```

### 4A.2 연속 분석 시나리오 (@important)

```
TC-4A.2.1: 같은 데이터 → 다른 메서드로 재분석
  시나리오:
    1. t-test.csv로 독립표본 t-검정 실행 → 결과 확인
    2. "새 분석" 버튼 클릭 (new-analysis-btn)
    3. 같은 데이터가 유지된 상태에서 Step 2로 이동
    4. 다른 메서드 (Mann-Whitney) 선택
    5. 변수 재할당 → 분석
    6. 두 번째 결과 확인
  검증:
    - 데이터 재업로드 불필요
    - 이전 결과와 독립적인 새 결과

TC-4A.2.2: 분석 이력에서 과거 결과 복원
  시나리오:
    1. 분석 1회 수행 → 결과 저장
    2. Hub로 돌아감
    3. AnalysisHistoryPanel에서 이전 분석 클릭
    4. 이전 결과 복원 확인
  검증:
    - 이력 패널에 최근 분석 표시
    - 클릭 시 결과 복원 (메서드, 변수, 결과값)

TC-4A.2.3: 변수만 변경하여 재분석 (ReanalysisPanel)
  시나리오:
    1. ANOVA 분석 완료
    2. 재분석 패널에서 종속변수 변경
    3. 재분석 실행
    4. 새 결과 비교
  검증:
    - 메서드 재선택 불필요
    - 빠르게 재분석 가능
```

### 4A.3 에러 복구 시나리오 — 통계 (@critical)

```
TC-4A.3.1: 잘못된 변수 할당 → 에러 → 수정
  시나리오:
    1. t-test 선택 후 텍스트 변수를 종속변수에 할당 시도
    2. 에러/경고 메시지 확인
    3. 올바른 수치 변수로 재할당
    4. 정상 분석 완료
  검증:
    - 에러 메시지가 사용자 친화적 (한국어)
    - 복구 경로가 명확

TC-4A.3.2: Pyodide 로딩 실패 → 재시도
  시나리오:
    1. 네트워크 차단 상태에서 분석 시도
    2. Pyodide 로딩 실패 메시지
    3. 네트워크 복구 후 재시도 버튼 클릭
    4. 정상 분석
  검증:
    - 실패 시 명확한 안내
    - 재시도 버튼 존재
    - 페이지 새로고침 없이 복구 가능 여부

TC-4A.3.3: 분석 중 취소
  시나리오:
    1. 분석 실행 버튼 클릭
    2. 로딩 중 "취소" 또는 뒤로가기
    3. Step 3(변수 선택)으로 복귀
    4. 변수 수정 → 재실행
  검증:
    - 취소 후 앱 상태 정상
    - 이전 설정(데이터, 메서드) 유지

TC-4A.3.4: 데이터 부족 에러
  시나리오:
    1. 3행짜리 극소 CSV 업로드
    2. ANOVA 선택 → 그룹당 관측수 부족
    3. 에러 메시지 + 권장 대안 확인
  검증:
    - "표본 수가 부족합니다" 유사 메시지
    - 비모수 대안 제안 (있을 경우)
```

## Part B: 그래프/시각화 UX (ux/graph-ux.spec.ts)

### 4B.1 Graph Studio 첫 사용자 시나리오 (@critical)

```
TC-4B.1.1: 처음 Graph Studio 방문 → 빠른 차트 생성
  시나리오:
    1. /graph-studio 접근
    2. 차트 유형 썸네일 중 "bar" 클릭
    3. 샘플 데이터로 차트 자동 생성
    4. 차트 확인 → 사이드 패널에서 데이터 확인
  검증:
    - 업로드 없이 1클릭으로 차트 생성
    - 차트가 보기 좋게 렌더링

TC-4B.1.2: 자기 데이터로 차트 생성
  시나리오:
    1. /graph-studio → 파일 업로드 영역에 CSV 드래그
    2. 자동 차트 생성
    3. 데이터 탭에서 X/Y축 변수 확인
    4. 차트 유형 변경 (bar → line)
  검증:
    - 업로드 후 3초 내 차트 렌더링
    - 변수 매핑이 직관적

TC-4B.1.3: Smart Flow 결과에서 Graph Studio로 이동
  시나리오:
    1. Smart Flow에서 t-test 분석 완료
    2. 결과 화면의 "Graph Studio" 버튼 (open-graph-studio-btn) 클릭
    3. Graph Studio에 분석 데이터 자동 로드
    4. 차트 커스터마이징
  검증:
    - 데이터 재업로드 불필요
    - 분석에 사용된 변수가 자동 매핑
```

### 4B.2 차트 커스터마이징 시나리오 (@important)

```
TC-4B.2.1: 논문용 차트 — 흑백 + 패턴
  시나리오:
    1. Bar 차트 생성
    2. 스타일 탭 → 컬러 모드를 "흑백" 변경
    3. 그리드 라인 표시
    4. 폰트 크기 조정
  검증:
    - 차트가 학술 논문에 적합한 스타일로 변경
    - 변경 사항이 즉시 반영

TC-4B.2.2: 차트 제목 + 축 레이블 수정
  시나리오:
    1. 차트 제목 더블클릭 → 인라인 편집
    2. 또는 사이드 패널에서 제목 입력
    3. X축/Y축 레이블 수정
  검증:
    - 한국어 입력 정상 동작
    - 제목이 차트에 반영

TC-4B.2.3: Undo/Redo로 실수 복구
  시나리오:
    1. 차트 색상 변경 (원본: 파랑)
    2. 또 변경 (빨강)
    3. Undo (graphStudioUndo) → 파랑 복귀
    4. Undo → 원본 복귀
    5. Redo (graphStudioRedo) → 파랑
  검증:
    - 각 Undo/Redo 시 차트 즉시 반영
    - 5단계+ 히스토리 지원
```

## Part C: 공통 UX 시나리오 (ux/common-ux.spec.ts)

```

```

