# AI Export 전략: 연구 패키지 체계화

**Last updated**: 2026-04-03
**References**: [Product Strategy](PRODUCT_STRATEGY.md), [Roadmap](../ROADMAP.md)
**정기 점검**: 분기별 (다음: 2026-07) — 경쟁 환경, AI 모델 동향, 포지셔닝 유효성 확인

---

## 1. 핵심 방향

BioHub은 논문을 직접 쓰는 도구가 아니다.
**AI 모델(Gemini, ChatGPT, Claude)에 넘길 "연구 패키지"를 체계화하는 플랫폼**이다.

---

## 2. 3단계 논문 완성 모델

```
┌─────────────────────────────────────────────────────────┐
│  1차 BioHub (Gemini 2.5 Flash Lite)                     │
│  ─────────────────────────────────────                   │
│  • 검증된 라이브러리로 통계/그래프/계산 실행              │
│  • 결과 체계화: 파일, 그림, 표, 목차 정리                │
│  • AI-ready 패키지 export (Markdown/JSON)                │
│                                                         │
│  내부 AI 역할: 분석 추천, 결과 해석 보조, 차트 patch     │
│  딥리서치/논문 생성 안 함 → 비용 효율 극대화             │
├─────────────────────────────────────────────────────────┤
│  2차 사용자의 SOTA AI (Claude/GPT/Gemini Pro)            │
│  ─────────────────────────────────────                   │
│  • BioHub 패키지를 받아서 논문 초고 생성                 │
│  • 딥리서치, 문맥 이해, 학술 문체 적용                   │
│  • 사용자가 이미 구독 중인 모델 활용                     │
├─────────────────────────────────────────────────────────┤
│  3차 사람 (Word/HWP)                                     │
│  ─────────────────────────────────────                   │
│  • 학회 템플릿(HWPX/DOCX/LaTeX)에 맞춰 최종 편집        │
│  • 한국 학회 → HWP/HWPX, 국제 저널 → DOCX/LaTeX         │
│  • 제출                                                 │
└─────────────────────────────────────────────────────────┘
```

### 왜 BioHub이 직접 논문을 쓰지 않는가

- **비용**: Flash Lite로 체계화만 하면 API 비용 최소. 딥리서치급 작업은 사용자의 SOTA 모델에 위임
- **품질**: 논문 생성은 최고 성능 모델이 해야 품질이 나옴. BioHub이 중급 모델로 하면 오히려 가치 하락
- **유연성**: 사용자가 선호하는 AI를 자유롭게 선택 (Claude, GPT, Gemini 등)

### 왜 AI가 직접 통계를 대체할 수 없는가

SOTA AI가 코드를 짜도 **1%의 오류가 학술에서는 치명적**:
- `ttest_ind` vs `ttest_rel` 혼동 → p값 오류
- 자유도 계산 실수 → reviewer가 잡아냄
- 그래프 축 스케일 오류 → 결론 왜곡

연구자가 매번 AI 코드를 검증하는 비용 > BioHub의 검증된 라이브러리를 쓰는 비용.
**BioHub = hallucination 없는 분석 레이어**.

## 4. 보완 필요 영역 (우선순위순)

### 4-1. AI Export 기능 (1순위 — 핵심 차별화)

프로젝트의 모든 결과를 **하나의 구조화된 문서로 내보내기**.

- 통계 결과 + 그래프 + 문헌 리스트 + 계산 결과를 Markdown/JSON으로 export
- AI 모델이 바로 소비할 수 있는 포맷 (맥락 포함)
- 예시 출력:
  ```
  ## 연구 개요
  - 주제: 해양 생태계 종 다양성 비교
  - 데이터: 3개 해역, 120 샘플

  ## 통계 결과
  - ANOVA: F(2,117) = 4.23, p = 0.017 (Table 2용)
  - Post-hoc Tukey: A-B 유의차 (p = 0.012)

  ## 그래프
  - [Figure 1] 종 다양성 비교 박스플롯 (base64 또는 경로)

  ## 인용 문헌
  - Kim et al. (2024) — 방법론 근거
  - Park et al. (2025) — 비교 데이터
  ```

**이것이 없으면 BioHub은 "통계 도구 모음"에 불과하고, 이것이 있으면 "AI 시대 연구 허브"가 된다.**

### 4-2. 결과 ↔ 논문 섹션 매핑 (2순위)

- 각 분석 결과에 논문 섹션 태그 부여: `Table 2`, `Figure 3`, `Results 3.1`
- 논문 구조(Introduction → Methods → Results → Discussion)에 자동 배치
- AI에게 넘길 때 "이 데이터는 Results 섹션용"이라는 맥락 자동 포함

### 4-3. 프롬프트 템플릿 (3순위)

- 분야별 논문 작성 프롬프트 생성기
  - "이 통계 결과로 Results 섹션을 APA 스타일로 작성해줘"
  - "이 문헌 목록을 기반으로 Introduction의 선행연구 검토를 작성해줘"
- 생태학, 유전학, 수산학 등 분야별 템플릿
- 저널별 스타일 가이드 반영

### 4-4. Publication-ready 그래프 Export (4순위)

- 저널 투고용 포맷: TIFF 300dpi, EPS, SVG
- APA/학술 스타일 자동 적용 (축 라벨, 폰트 크기, 범례 위치)
- Figure 캡션 자동 생성 (분석 메타데이터 기반)

### 4-5. 버전/히스토리 비교 (5순위)

- 분석 조건 변경 시 결과 차이 추적
- Reviewer 코멘트 대응: "분석 조건을 X로 바꾸면 결과가 어떻게 바뀌는지" 즉시 비교
- 분석 재현성 보장

## 5. 시장성 평가

### 긍정 요인

- **타이밍**: 연구자의 AI 활용 폭발적 증가 (2024~2026)
- **공백**: 기존 통계 SW(SPSS, R, JASP)는 AI 연동 없음
- **블루오션**: "AI-ready research workflow" 카테고리 자체가 신규
- **비용 우위**: Cloudflare 기반 → 낮은 운영비, 무료/저가 진입 가능
- **프라이버시**: 브라우저 내 계산 → 민감한 연구 데이터 외부 전송 불필요

### 위험 요인

- **AI 직접 통계**: ChatGPT/Claude가 통계+그래프 기능을 강화하면 중간 레이어 가치 감소
  - 단, **1% 오류도 허용 안 되는 학술 영역**에서 검증된 라이브러리 기반 BioHub의 신뢰성이 해자
  - 도메인 특화 워크플로우 + 히스토리 + 체계화 + 재현성은 범용 AI가 대체 어려움
- **타겟 집중 필요**: "모든 연구자" → **"생물학/생태학/수산학 연구자"**로 좁혀야 함

### 왜 BioHub의 포지션이 비어 있는가

기존 도구들은 **한 가지만 잘한다**:

```
통계 도구 (JASP, SPSS)     → 분석은 하지만 AI 연동 없고 export는 복붙
AI 작성 도구 (Paperpal 등) → 글은 쓰지만 통계 엔진 없음
코드 기반 (Quarto)         → 둘 다 하지만 코딩 필수 → 비전공자 진입장벽
문헌 관리 (Zotero 등)      → 레퍼런스만, 분석/작성 없음
```

**"GUI로 통계 → 결과 체계화 → AI가 소비 가능한 패키지 export"를 하는 도구는 현재 없다.**

이 공백이 존재하는 이유:
1. 통계 도구 회사(IBM, GraphPad)는 AI 연동에 관심 없음 (레거시 수익 모델)
2. AI 작성 도구(Paperpal 등)는 통계 엔진을 만들 역량/의지 없음
3. Quarto/R Markdown은 코딩 전제라 GUI 시장과 다른 세그먼트
4. 한국 시장은 HWP 중심이라 글로벌 도구가 진입 안 함

**BioHub의 "검증된 분석 + 체계화 + AI Export" 조합은 현재 직접 경쟁자 없음.**

### 끼어들 수 있는 경쟁 지점

| 구간 | 위협 후보 | 위협도 | 이유 |
|------|----------|--------|------|
| 1차 (분석) | AI Code Interpreter | 중 | 오류 위험으로 학술 신뢰 부족 |
| 1차→2차 (패키지 전달) | 없음 | **낮음** | BioHub 핵심 해자 |
| 2차 (초고 생성) | ChatGPT/Claude/Gemini | 위협 아님 | BioHub이 이들을 활용하는 구조 |
| 2차→3차 (포맷 변환) | Typst, SciSpace | 낮음 | 한국 학회는 HWP 복붙이 현실 |
| 3차 (최종 편집) | Word/HWP | 위협 아님 | BioHub 영역 밖 |

## 6. 실행 우선순위

```
Phase 현재: 통계 엔진 + Bio-Tools + 논문 템플릿 (완료/진행 중)
     ↓
Phase A: AI Export 기능 (구조화된 연구 패키지 내보내기)
     ↓
Phase B: 결과 ↔ 논문 섹션 매핑 + 태깅
     ↓
Phase C: 프롬프트 템플릿 (분야별 논문 작성 가이드)
     ↓
Phase D: Publication-ready 그래프 export
     ↓
Phase E: 버전/히스토리 비교 (Reviewer 대응)
```

## 7. 결론

BioHub의 방향은 적절하며 타이밍도 맞다.

**BioHub은 논문을 쓰는 도구가 아니라, 논문을 쓸 수 있게 만드는 도구다.**

- 1차(BioHub): 검증된 분석 + 체계화 → 저비용 Flash Lite로 충분
- 2차(SOTA AI): 사용자의 최고 모델로 초고 생성 → BioHub이 만든 패키지가 품질 결정
- 3차(사람): 학회 템플릿에 맞춰 최종 제출

핵심은 **AI Export** — 이 기능이 BioHub의 존재 이유가 된다.
