```
2026: 무료 배포 + 사용자 확보 + 데이터 축적
2027: 공모전 + 정부지원 + 수익화
```

---

## 2. 배포 원칙

| 원칙 | 설명 |
|------|------|
| 완벽하게 만들고 배포 | **하지 않는다** |
| 사용할 수 있는 수준이면 바로 배포 | **이것이 기본** |
| 무료로 풀되, 무제한 제공 | **하지 않는다** |
| 제한된 무료(Freemium) 구조 | **이것이 기본** |

---

## 3. MVP 배포 범위

### 포함 (필수)

- 로그인 / 사용자 식별
- 핵심 기능 1-2개
- 기본 결과 출력
- 사용 로그 수집

### 제외 (초기에는 안 함)

- 완벽한 UI
- 모든 기능 구현
- 고급 분석 기능

---

## 4. Freemium 구조

### 무료 티어

- 하루 사용 제한 (예: 10회)
- 기본 결과만 제공
- 일부 기능 제한

### 유료 티어 (추후 오픈)

- 고급 분석 결과
- 대량 처리
- 리포트 생성
- API 제공

> 상세 가격 구조: [monetization/ROADMAP.md](../monetization/ROADMAP.md) 참조

---

## 5. 타겟 사용자

### 타겟

- 연구자
- 공무원
- 실무 분석 사용자

### 하지 말 것

- 일반 대중 대상 확장
- 무작위 사용자 유입

---

## 6. 데이터 수집 설계

**반드시 수집할 항목:**

| 항목 | 목적 |
|------|------|
| 검색 키워드 | 기능 개선 방향 |
| 사용 기능 | 우선순위 판단 |
| 실패 케이스 | 버그 수정 + UX 개선 |
| 체류 시간 | 기능별 사용성 평가 |
| 반복 사용 여부 | 핵심 기능 식별 |

**활용 목적:**

- 기능 개선
- DB 고도화
- AI 성능 향상

> 기술 구현: Cloudflare Workers + D1 로그 테이블
> 분석 도구: GA4 + Clarity (스택 참조: `d:\Projects\dev-playbook\stack\analytics.md`)

## 7. 성공 지표 (2026 목표)

| 지표 | 목표 |
|------|------|
| 사용자 수 | 500-2,000명 |
| 실제 사용 사례 | 10개 이상 |
| 반복 사용자 | 확보 (재방문율 추적) |
| 데이터 누적 | 기능 개선에 활용 가능한 수준 |

---

## 8. 운영 전략

1. 무료로 빠르게 배포
2. 사용자 피드백 즉시 반영
3. 데이터 기반 기능 개선
4. 핵심 기능 위주로 반복 개선

---

## 9. 절대 하지 말 것

| 금지 사항 | 이유 |
|-----------|------|
| 완벽주의로 배포 지연 | 시장 검증이 먼저 |
| 핵심 로직 공개 | 경쟁 우위 보호 |
| 데이터 수집 없이 운영 | 개선 근거 확보 불가 |
| 무료 무제한 제공 | 서버 비용 + 수익화 기반 훼손 |

---

## 10. 최종 목표

```
사용자 기반 확보
  → 데이터 기반 경쟁력 구축
    → 공모전/정부지원에서 "이미 검증된 서비스"로 포지셔닝
```
