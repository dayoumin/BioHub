## Design Context

### Users

- **누구**: 수산과학 연구자, 통계 전문가, 데이터 분석가 (SPSS/R Studio 대안 사용자)
- **맥락**: 일상적 업무 — 데이터 수집 후 분석, 논문용 통계 처리, 반복적 루틴 작업
- **목표**: 복잡한 통계 분석을 코드 없이 정확하고 빠르게 수행
- **감성 목표**: 신뢰감 — "이 도구가 내놓은 결과를 믿을 수 있다"는 확신

### Brand Personality

- **3단어**: 전문적, 깔끔한, 신뢰할 수 있는
- **톤**: 차분하고 절제된 과학적 톤. 화려함보다 명확함 우선
- **레퍼런스**: Perplexity AI (클린 인터페이스, 정보 밀도, 미니멀 구조)
- **안티-레퍼런스**: 미정 — 추후 추가

### Aesthetic Direction

- **시각적 톤**: 미니멀 + 전문적. 과학 도구답게 데이터와 결과에 시선 집중
- **색상 체계**: OKLch 기반 Science Blue (#2563eb) 주색상, 섹션별 accent 분리 (analysis=teal, bio=green, graph=violet)
- **서체**: Pretendard Variable (본문) + JetBrains Mono (코드/수치)
- **다크모드**: 지원 (next-themes), CSS 변수 자동 전환
- **포인트**: 글래스모피즘 포인트, 스프링 기반 마이크로 인터랙션
- **현재 과제**: 통계 컴포넌트의 레이아웃/디자인이 전체 시스템과 일관성 있는지 검증 필요

### Design Principles

1. **신뢰 우선 (Trust First)** — 데이터, 수치, 결과가 정확하고 읽기 쉬워야 한다. 장식보다 명확성
2. **일관된 절제 (Consistent Restraint)** — 모든 섹션이 같은 토큰, 같은 간격, 같은 타이포그래피 계층을 따른다
3. **데이터가 주인공 (Data Takes Center Stage)** — UI는 데이터를 돋보이게 하는 프레임. 차트와 테이블이 시각적 우선순위
4. **학술적 관례 존중 (Respect Academic Conventions)** — 유의성 색상(초록=유의), 효과크기 해석, 통계 표기법은 학술 논문 관례를 따른다
5. **점진적 공개 (Progressive Disclosure)** — 복잡한 옵션은 단계별로 노출. 초보자도 핵심 흐름을 따라갈 수 있어야 한다

### Technical Foundation

- **프레임워크**: Next.js 15 + React 19 + TypeScript 5.7
- **컴포넌트**: shadcn/ui (new-york style, neutral base)
- **스타일링**: Tailwind CSS v4 + CSS variables (OKLch)
- **애니메이션**: Framer Motion 12 (spring/tween) + tw-animate-css
- **차트**: ECharts + Plotly + Vega (런타임 색상 해석, chart-color-resolver)
- **디자인 토큰**: `lib/design-tokens/` — common, analysis, bio, status, step-flow
- **접근성 목표**: WCAG AA
- **접근성 현황**: prefers-reduced-motion 지원, 포커스 링, 차트 마커 형태 구분 (색각 이상 대응)
