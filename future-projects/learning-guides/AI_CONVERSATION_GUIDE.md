# AI와 효과적으로 대화하는 방법

**작성일**: 2024-11-18
**대상**: AI 챗봇을 더 잘 활용하고 싶은 모든 분
**목표**: 프롬프트 엔지니어링 기초를 익혀 원하는 답변을 정확하게 받기

---

## 🎯 핵심 원칙

**"구체적으로, 명확하게, 예시와 함께"**

AI는 마음을 읽을 수 없습니다. 내가 원하는 걸 명확하게 말해야 합니다.

---

## 📋 좋은 질문 vs 나쁜 질문

### 예시 1: 개념 설명 요청

| 나쁜 질문 | 좋은 질문 |
|----------|----------|
| "RAG가 뭐야?" | "RAG를 초보자에게 설명한다면? Google 검색과 비교해서, 내부망 환경에서 왜 유리한지 포함해서" |
| "임베딩이 뭐야?" | "임베딩을 비유로 설명해줘. '단어를 숫자로 바꾼다'는 말이 이해 안 돼" |
| "프롬프트 체이닝?" | "프롬프트 체이닝을 TypeScript 코드로 보여줘. RAG → 판단 → 실행 3단계로" |

**왜 좋은 질문인가?**
- ✅ 대상 명시 ("초보자에게")
- ✅ 비교 요청 ("Google 검색과 비교")
- ✅ 맥락 제공 ("내부망 환경")
- ✅ 형식 지정 ("TypeScript 코드로")

---

### 예시 2: 코드 요청

| 나쁜 질문 | 좋은 질문 |
|----------|----------|
| "ReactFlow 어떻게 써?" | "나는 React 초보자야. ReactFlow로 노드 2개를 연결하는 완전한 코드를 줘. 복붙해서 바로 실행할 수 있게" |
| "JSON 저장하는 법" | "JavaScript로 JSON 객체를 파일로 저장하는 코드를 줘. Blob과 download 속성 사용. 브라우저에서 동작해야 해" |
| "Ollama API 호출" | "curl 명령어로 Ollama API에 'RAG를 한 줄로 설명해줘' 질문을 보내는 예제를 줘" |

**왜 좋은 질문인가?**
- ✅ 내 수준 명시 ("React 초보자")
- ✅ 구체적 요구 ("노드 2개 연결")
- ✅ 실행 환경 지정 ("브라우저에서")
- ✅ 형식 지정 ("curl 명령어로")

---

### 예시 3: 문제 해결

| 나쁜 질문 | 좋은 질문 |
|----------|----------|
| "에러 났어" | "이 에러가 왜 나는지 설명해줘: `TypeError: Cannot read property 'map' of undefined at line 42`" |
| "코드가 안 돼" | "이 코드를 실행하면 예상과 다르게 동작해:\n\n```typescript\n[코드 붙여넣기]\n```\n\n예상: A가 출력\n실제: B가 출력\n\n왜 그럴까?" |
| "이해 안 돼" | "이 코드를 초보자에게 설명한다면?\n\n```typescript\n[코드 붙여넣기]\n```\n\n특히 `useCallback`이 왜 필요한지 모르겠어" |

**왜 좋은 질문인가?**
- ✅ 에러 메시지 전체 제공
- ✅ 예상 vs 실제 비교
- ✅ 코드 전체 제공
- ✅ 모르는 부분 명확히 지정

## 🛠️ 프롬프트 엔지니어링 기법

### 기법 1: 역할 부여 (Role Prompting)

```
❌ "RAG를 설명해줘"

✅ "너는 초보 개발자를 가르치는 친절한 선생님이야.
   RAG를 비유를 들어 설명해줘."
```

**효과**: AI가 설명 스타일을 조정합니다.

---

### 기법 2: 단계별 지시 (Step-by-Step)

```
❌ "프로세스 빌더 만드는 법"

✅ "프로세스 빌더를 만드는 과정을 5단계로 나눠서 설명해줘.
   각 단계마다:
   1. 무엇을 하는지
   2. 필요한 기술
   3. 예상 시간
   을 포함해서"
```

**효과**: 구조화된 답변을 받을 수 있습니다.

---

### 기법 3: 예시 제공 (Few-Shot)

```
❌ "체크리스트 컴포넌트 만들어줘"

✅ "이런 형태의 체크리스트 컴포넌트를 만들어줘:

   예시:
   ☐ 입찰 공고문 작성
   ☑ 예산 범위 확인
   ☐ 법무팀 사전 검토

   요구사항:
   - useState로 상태 관리
   - 클릭하면 체크 토글
   - TypeScript 사용"
```

**효과**: 내가 원하는 형태를 정확히 전달합니다.

---

### 기법 4: 제약 조건 명시 (Constraints)

```
❌ "자동화 아이디어 줘"

✅ "내부망 환경에서 가능한 자동화 아이디어를 줘.

   제약 조건:
   - 인터넷 연결 없음
   - 로컬 Ollama만 사용 가능
   - 브라우저 기반 (HTML/JavaScript)
   - 외부 API 호출 불가

   5개 아이디어를 제안해줘."
```

**효과**: 실현 불가능한 답변을 방지합니다.

---

### 기법 5: 형식 지정 (Format)

```
❌ "통계 방법들 알려줘"

✅ "통계 방법 10개를 표로 정리해줘.

   | 방법 | 용도 | 예시 |
   |------|------|------|
   | t-검정 | 평균 비교 | 신약 효과 검증 |

   이런 형식으로"
```

**효과**: 읽기 쉬운 형태로 답변을 받습니다.

## 🎯 맥락(Context) 제공하기

AI는 내 상황을 모릅니다. 반드시 알려줘야 합니다!

### 제공할 맥락 목록

1. **내 수준**
   - "나는 JavaScript 초보자야"
   - "React는 할 줄 알지만 TypeScript는 처음이야"

2. **현재 상황**
   - "현재 통계 플랫폼 프로젝트를 진행 중이야"
   - "RAG 시스템은 이미 구현되어 있어"

3. **제약 조건**
   - "인터넷 연결 없는 내부망이야"
   - "로컬 Ollama만 사용 가능해"

4. **목표**
   - "2개월 내에 프로토타입 만들기"
   - "하루 30분씩만 투자 가능"

5. **환경**
   - "Windows 10 사용 중"
   - "Chrome 브라우저만 지원하면 돼"

## 💡 고급 팁

### Tip 1: 비교 요청하기

```
❌ "ReactFlow가 뭐야?"
✅ "ReactFlow vs React-DnD 비교해줘. 프로세스 빌더에는 어느 게 적합해?"
```

### Tip 2: 장단점 물어보기

```
❌ "로컬 Ollama 쓰면 돼?"
✅ "로컬 Ollama의 장단점을 알려줘. 클라우드 API(ChatGPT)와 비교해서"
```

### Tip 3: 대안 요청하기

```
❌ "이 방법으로 해야 해?"
✅ "이 방법 말고 다른 방법 2가지를 제안해줘. 각각의 장단점도 포함해서"
```

### Tip 4: 트레이드오프 확인하기

```
❌ "성능 개선하려면?"
✅ "성능 vs 코드 복잡도 트레이드오프를 고려해서 3가지 개선 방법을 제안해줘"
```
