## 🎯 핵심 개념

### 문제
- 각 부서(시설, 예산, 계약, 인사 등)가 **각자의 문서**(규정, 참고문헌, 부록)를 가지고 있음
- 기존 방식: 모든 부서 문서를 **하나의 DB에 통합** → 느린 검색 + 권한 관리 복잡

### 해결
- 각 부서가 **독립적인 DB 생성** (예: `rag-facility.db`)
- **파일 서버에 공유** → 다른 직원이 다운로드 → 즉시 사용
- **UI에서 DB 선택** → 실시간 전환

## ⚡ 성능

| 항목 | 통합 DB (8개 부서) | 독립 DB (1개 부서) |
|------|-------------------|-------------------|
| 문서 수 | 2,408개 | 301개 |
| 청크 수 | 38,720개 | 4,840개 |
| 검색 속도 | ~1.94초 ❌ | ~0.24초 ✅ |
| 메모리 | ~468 MB | ~58 MB |

→ **독립 DB가 8배 빠름!**
