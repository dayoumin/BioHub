# Process + RAG 시스템

**업무 프로세스 빌더 + RAG 통합 플랫폼**

---

## 🎯 프로젝트 개요

**문제**:
- 신입 직원이 계약/시설/예산 등 복잡한 업무 절차를 파악하기 어려움
- 업무 절차가 수시로 변경되어 최신 정보 유지 어려움
- 각 부서별로 업무 가이드가 따로 관리되어 접근성 낮음

**해결**:
- 업무 담당자가 **드래그앤드롭**으로 프로세스 생성
- JSON 파일로 저장 → **파일 서버 공유** (RAG DB와 동일)
- 다른 직원이 다운로드 → 개인 PC에서 실행 (체크/메모)
- 각 단계마다 **RAG 챗봇**으로 질문 가능

---

## 📋 주요 기능

### 1️⃣ 프로세스 빌더 (담당자용)
- ✅ 드래그앤드롭으로 단계 추가
- ✅ 체크리스트 자동 생성
- ✅ 파일 첨부 슬롯
- ✅ 조건 분기 (Yes/No)
- ✅ JSON 파일 저장

### 2️⃣ 프로세스 실행기 (직원용)
- ✅ 단계별 가이드 표시
- ✅ 체크박스 (완료 표시)
- ✅ 메모 (개인별 기록)
- ✅ 파일 업로드
- ✅ 진행률 추적

### 3️⃣ RAG 통합
- ✅ 각 단계마다 질문 가능
- ✅ 부서별 RAG DB 자동 연결
- ✅ 컨텍스트 자동 생성

### 4️⃣ 공유 시스템
- ✅ JSON 파일 공유 (파일 서버)
- ✅ 버전 관리 (v1.0, v1.1, ...)
- ✅ 프로세스 레지스트리 (중앙 목록)

---

## 🚀 사용 시나리오

### A. 담당자 (프로세스 생성)
```
1. 프로세스 빌더 실행
2. 단계 추가 (드래그앤드롭)
   - 입찰 공고 작성
   - 법무 검토
   - 계약 체결
3. 각 단계에 체크리스트 추가
4. "저장" → contract-process-v1.2.json
5. 파일 서버 업로드
```

### B. 직원 (프로세스 실행)
```
1. 파일 서버에서 다운로드
2. 프로세스 실행기 열기
3. 단계별로 따라하기
   - ☑ 입찰 공고문 작성
   - ☑ 예산 범위 확인
   - 📝 메모: "예산 1억 원 초과"
4. 각 단계에서 RAG 질문
   - "법무 검토 기준이 뭐야?"
```

---

## 📊 시스템 구조

```
프로세스 빌더 (ReactFlow)
    ↓
JSON 파일 (contract-process-v1.2.json)
    ↓
파일 서버 (//fileserver/shared/processes/)
    ↓
프로세스 실행기 (개인 PC)
    ↓
RAG 챗봇 (부서별 DB 자동 연결)
```
