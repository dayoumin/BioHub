핵심 원칙
• 단건 분석에 마찰 없음 — 연구과제 없이도 모든 기능 동일하게 사용
• 자연스러운 전환 — 분석이 쌓이면 연구과제의 가치를 느끼게 해줌
• 비침입적 — 넛지는 최초 1회만, 거부해도 다시 안 뜸
• 즉각적 피드백 — 활성 연구과제가 있으면 저장 토스트에 과제명 표시
• 히스토리 = 연구과제 진입점 — 어디서든 연구과제로 이동/추가 가능
• 메뉴 간소화 — "연구 프로젝트" 메뉴 삭제, 드롭다운 ⚙ 한 곳에서 관리
